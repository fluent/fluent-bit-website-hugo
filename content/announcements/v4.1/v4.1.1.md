###### KNOWLEDGE BASE

## Release Notes v4.1.1

Fluent Bit is a Fast and Lightweight Data Processor and Forwarder for Linux, BSD, Windows and macOS. We are proud to announce the availability of Fluent Bit v4.1.1.

Below is a detailed list of changes included in this release...


### What's new ?

<br>

#### Core

<br>

- opentelemetry: logs: skip empty strings for TraceID and SpanID ([PR #10956](https://github.com/fluent/fluent-bit/pull/10956))
- reload: refactor watchdog to be preemptable on windows. ([PR #10968](https://github.com/fluent/fluent-bit/pull/10968))
- router: do logic matching using tag length only ([PR #10976](https://github.com/fluent/fluent-bit/pull/10976))

### Plugins

<br>

- **[Emitter (Input)](https://docs.fluentbit.io/manual/pipeline/inputs/emitter)**
  - Resolve ring buffer gtliches on threaded mode with multiline parser ([PR #10950](https://github.com/fluent/fluent-bit/pull/10950))
- **[Forward (Input)](https://docs.fluentbit.io/manual/pipeline/inputs/forward)**
  - Fix user authentication mechanism to align Fluentd behavior ([PR #10973](https://github.com/fluent/fluent-bit/pull/10973))
  - Add a missing releasing users related contexts ([PR #10973](https://github.com/fluent/fluent-bit/pull/10973))
  - Handle shared_key lifetime correctly ([PR #10973](https://github.com/fluent/fluent-bit/pull/10973))
- **[Splunk (Input)](https://docs.fluentbit.io/manual/pipeline/inputs/splunk)**
  - return correct error response for HTTP/1 handler. ([PR #10940](https://github.com/fluent/fluent-bit/pull/10940))
  - propagate the return value inside process_hec_payload from handle_hec_payload. ([PR #10940](https://github.com/fluent/fluent-bit/pull/10940))
  - use record accessor for tag key extraction and add listening info ([PR #10967](https://github.com/fluent/fluent-bit/pull/10967))
- **[Winevtlog (Input)](https://docs.fluentbit.io/manual/pipeline/inputs/winevtlog)**
  - Try to reconnect on cancellations ([PR #10942](https://github.com/fluent/fluent-bit/pull/10942))
  - Add retry backoff mechanism ([PR #10942](https://github.com/fluent/fluent-bit/pull/10942))
  - Handle capped values ([PR #10942](https://github.com/fluent/fluent-bit/pull/10942))
  - Plug NULL pointer deref lines ([PR #10942](https://github.com/fluent/fluent-bit/pull/10942))
  - Process remote handle to retrive metadata if available ([PR #10942](https://github.com/fluent/fluent-bit/pull/10942))
  - Plug a negative overflow possibility ([PR #10942](https://github.com/fluent/fluent-bit/pull/10942))
  - Apply backoff settings effectively ([PR #10942](https://github.com/fluent/fluent-bit/pull/10942))
- **[Docker (Input)](https://docs.fluentbit.io/manual/pipeline/inputs/docker)**
  - add helper for container name parsing ([PR #10972](https://github.com/fluent/fluent-bit/pull/10972))
- **[HTTP (Input)](https://docs.fluentbit.io/manual/pipeline/inputs/http)**
  - fix handling of URL encoded value ([PR #10961](https://github.com/fluent/fluent-bit/pull/10961))
  - use record accessor for tag key extraction ([PR #10967](https://github.com/fluent/fluent-bit/pull/10967))
- **[Elasticsearch (Input)](https://docs.fluentbit.io/manual/pipeline/inputs/elasticsearch)**
  - reimplement `tag_key` lookup with record accessor api ([PR #10967](https://github.com/fluent/fluent-bit/pull/10967))
- **[Calyptia_fleet (Input)](https://docs.fluentbit.io/manual/pipeline/inputs/calyptia_fleet)**
  - load configuration on initial startup. ([PR #10996](https://github.com/fluent/fluent-bit/pull/10996))
  - resume main collector if initial collector fails to schedule. ([PR #10996](https://github.com/fluent/fluent-bit/pull/10996))
  - consistent cleanup when failing to initliaze initial one time collector. ([PR #10996](https://github.com/fluent/fluent-bit/pull/10996))
- **[Opentelemetry (Output)](https://docs.fluentbit.io/manual/pipeline/outputs/opentelemetry)**
  - add configurable log resource limits ([PR #10955](https://github.com/fluent/fluent-bit/pull/10955))
  - add configurable log scope limits ([PR #10955](https://github.com/fluent/fluent-bit/pull/10955))
  - no longer reallocates the http2 setting string ([PR #10960](https://github.com/fluent/fluent-bit/pull/10960))
- **[File (Output)](https://docs.fluentbit.io/manual/pipeline/outputs/file)**
  - sanitize incoming Tag to prevent path traversal ([PR #10969](https://github.com/fluent/fluent-bit/pull/10969))
- **[Elasticsearch (Output)](https://docs.fluentbit.io/manual/pipeline/outputs/es)**
  - create space for header before use ([PR #10949](https://github.com/fluent/fluent-bit/pull/10949))
