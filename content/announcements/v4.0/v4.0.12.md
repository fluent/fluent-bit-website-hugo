###### KNOWLEDGE BASE

## Release Notes v4.0.12

Fluent Bit is a Fast and Lightweight Data Processor and Forwarder for Linux, BSD, Windows and macOS. We are proud to announce the availability of Fluent Bit v4.0.12.

Below is a detailed list of changes included in this release...


### What's new ?

<br>

#### Core

<br>

- router: do logic matching using tag length only ([PR #10987](https://github.com/fluent/fluent-bit/pull/10987))
- opentelemetry: logs: skip empty strings for TraceID and SpanID ([PR #10962](https://github.com/fluent/fluent-bit/pull/10962))
- reload
  - implement watchdog thread for hot reload timeout ([PR #10985](https://github.com/fluent/fluent-bit/pull/10985))
  - refactor watchdog to be preemptable on windows. ([PR #10988](https://github.com/fluent/fluent-bit/pull/10988))
- tests
  - internal: opentelemetry: add tests for empty traceid/spanid ([PR #10962](https://github.com/fluent/fluent-bit/pull/10962))
  - add hot reload watchdog timeout test ([PR #10985](https://github.com/fluent/fluent-bit/pull/10985))
  - internal: router: add edge cases for matching rules ([PR #10987](https://github.com/fluent/fluent-bit/pull/10987))
- config: add Hot_Reload.Timeout configuration option ([PR #10985](https://github.com/fluent/fluent-bit/pull/10985))

### Plugins

<br>

- **[Splunk (Input)](https://docs.fluentbit.io/manual/pipeline/inputs/splunk)**
  - return correct error response for http/1 handler. ([PR #10939](https://github.com/fluent/fluent-bit/pull/10939))
  - add record accessor support for tag key extraction and add listening info ([PR #10991](https://github.com/fluent/fluent-bit/pull/10991))
- **[Emitter (Input)](https://docs.fluentbit.io/manual/pipeline/inputs/emitter)**
  - Resolve ring buffer gtliches on threaded mode with multiline parser ([PR #10951](https://github.com/fluent/fluent-bit/pull/10951))
- **[Forward (Input)](https://docs.fluentbit.io/manual/pipeline/inputs/forward)**
  - Fix user authentication mechanism to align Fluentd behavior ([PR #10986](https://github.com/fluent/fluent-bit/pull/10986))
  - Add a missing releasing users related contexts ([PR #10986](https://github.com/fluent/fluent-bit/pull/10986))
  - Handle `shared_key` lifetime correctly ([PR #10986](https://github.com/fluent/fluent-bit/pull/10986))
- **[HTTP (Input)](https://docs.fluentbit.io/manual/pipeline/inputs/http)**
  - fix handling of URL encoded value ([PR #10982](https://github.com/fluent/fluent-bit/pull/10982))
  - add record accessor support for tag key extraction ([PR #10991](https://github.com/fluent/fluent-bit/pull/10991))
- **[Docker (Input)](https://docs.fluentbit.io/manual/pipeline/inputs/docker)**
  - add helper for container name parsing ([PR #10984](https://github.com/fluent/fluent-bit/pull/10984))
- **[Calyptia_fleet (Input)](https://docs.fluentbit.io/manual/pipeline/inputs/calyptia_fleet)**
  - Unify fleet config file caching across platforms ([PR #10985](https://github.com/fluent/fluent-bit/pull/10985))
  - Only commit fleet config when successfully reloaded ([PR #10985](https://github.com/fluent/fluent-bit/pull/10985))
- **[Dummy (Input)](https://docs.fluentbit.io/manual/pipeline/inputs/dummy)**
  - add test flag for hot reload timeout testing ([PR #10985](https://github.com/fluent/fluent-bit/pull/10985))
- **[Elasticsearch (Input)](https://docs.fluentbit.io/manual/pipeline/inputs/elasticsearch)**
  - reimplement `tag_key` lookup with record accessor api ([PR #10991](https://github.com/fluent/fluent-bit/pull/10991))
- **[Opentelemetry (Output)](https://docs.fluentbit.io/manual/pipeline/outputs/opentelemetry)**
  - no longer reallocates the http2 setting string ([PR #10963](https://github.com/fluent/fluent-bit/pull/10963))
  - add configurable log resource limits ([PR #10981](https://github.com/fluent/fluent-bit/pull/10981))
  - add configurable log scope limits ([PR #10981](https://github.com/fluent/fluent-bit/pull/10981))
- **[File (Output)](https://docs.fluentbit.io/manual/pipeline/outputs/file)**
  - sanitize incoming Tag to prevent path traversal ([PR #10983](https://github.com/fluent/fluent-bit/pull/10983))
- ** [Log to Metrics (Filter)](https://docs.fluentbit.io/manual/data-pipeline/filters/log_to_metrics)**
  - Fix improper handling of record accessor context ([PR #10980](https://github.com/fluent/fluent-bit/pull/10980))
